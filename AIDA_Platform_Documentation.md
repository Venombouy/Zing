#### 5.2.6 Specification (System Architecture)
Describes AIDA's backend as modular, scalable, and blockchain-agnostic, in **6 layers**:

1. **Blockchain Layer** — entry point per network. Components:
   - *Basic Client* (blocks, transactions, latest block)
   - *DEX Adapter* (parses trading events: liquidity pools, order books, trade logs)
   - *Verifier* (validates cryptographic signatures across chains/standards)
   - *Internal Swap Adapter* (optional — validates internal swaps, referral logic)
2. **Scanner Layer** — extracts/processes raw chain data continuously.
   - *Block Scanner* — processes block-by-block, pushes parsed data into per-DEX queues
   - *Interval Scanner* — periodic fetch (e.g., liquidity changes on EVM), enriches Block Scanner data
3. **Data Processing Layer** — transforms raw data into structured form.
   - Data transformation → standardized format
   - Storage updates (PostgreSQL + Redis)
   - Cache preparation for real-time analytics/UI
   - Concurrency model: each queue processed by an independent thread
4. **Trade (Terminal) Layer** — handles trading data storage/aggregation/notifications.
   - Redis Trade Storage (in-memory, Lua-script atomic updates)
   - Hourly Aggregates (recalculated hourly; re-scans don't double-count)
   - Notification System (WebSocket broadcasts for real-time chart/pair updates)
   - New trades → durable storage in PostgreSQL; hot data in Redis
5. **Token Audit** — automated validation/integrity checks via external audit providers.
   - Flow: Selection → Queuing → Validation → Results storage/display
   - Modular — new audit providers can be added
6. **Swap Logic** — manages swap processing & referral rewards.
   - Flow: Initiation (API hook) → 10-minute confirmation delay → Validation & Storage (fee calc, referral rewards) → real-time WebSocket notification

**Adding New Networks** requires implementing: Basic Client, DEX Adapter, Verifier, and (optionally) Internal Swap Adapter — designed so new chains integrate without affecting existing components.

### 5.7 Section: MindShare (a.k.a. "Social Booster" / MindSharing)

#### 5.7.1 What is MindShare?
Campaign engine turning attention into adoption — projects set rules, KOLs/communities create traction, payouts happen instantly with a transparent leaderboard.

**Why MindSharing vs. classic KOL marketing:**
- Pay for real outcomes (engagement/quality/relevance), not follower count
- Instant payouts in liquid tokens — no lockups/vesting/invoices
- Cheaper/faster go-to-market — skip six-figure CEX listing budgets; just a reward pool + **5% platform fee**
- Works Pre-TGE or Post-TGE, funded in any supported token
- Fully transparent: live submission feed, on-chain accounting, public leaderboard
- AI-assisted quality control (100+ signal checks + "X-Score" to reduce spam/boost ROI)
- Optional Quests (follow/comment/retweet/join Telegram/Discord) to extend reach

**How it works:** Create campaign (define goals/rules/KOL-community balance/quests) → Fund reward pool (choose token/chain, deposit + 5% fee) → Go Live (KOLs/community post and get paid instantly, tracked in real time)

**For Projects:** outcome-based spend, no heavy upfront cost, Pre/Post-TGE ready, precision controls, real-time analytics, integrates with AIDA trading terminal & Trading Competitions.

**For KOLs & Creators:** instant token rewards, fair earning via X-Score + traction, multiple income streams (posts/quests/boosts), ongoing deal flow across chains/sectors.
